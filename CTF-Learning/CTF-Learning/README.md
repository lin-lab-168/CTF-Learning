一、漏洞原理
为什么输入 `1' or '1'='1' -- ` 会返回所有数据
（1'为用户信息，真值；'1'='1' -- 永真，or在里面返回到sql语句中有whether，所以此条件恒为真，所以会返回所有数据。）
输入 `1' or '1'='1' -- ` 后，后台实际执行的 SQL 语句变成：
SELECT first_name, last_name FROM users WHERE user_id = '1' or '1'='1' -- '
解释：
- `1'` 闭合了原来的单引号
- `or` 是“或”逻辑，两边只要有一个为真，整个条件就为真
- `'1'='1'` 是永真条件（字符串1永远等于字符串1）
- `-- ` 注释掉了后面多余的代码
因为 `or '1'='1'` 永远为真，所以 WHERE 条件永远成立，数据库返回了 users 表中的所有数据。

二、HTTP 知识回顾
GET 和 POST 的区别，参数放哪里，状态码 302 什么意思?
（GET是请求指定的资源，参数放在请求请求路径的后面，POST是向服务器提交数据，以创建或更新资源，参数放在请求体中。
状态码302是说资源临时被移到了另一个网址中。）

三、Burp 抓包验证（抓登录包的操作步骤）
理论上，抓登录包的流程是这样的：
1.打开 Burp，确保 Proxy → Intercept 中的拦截开关是 Intercept is on（红色状态）
在浏览器中设置代理为 127.0.0.1:8080，或用 Burp 自带的 Open Browser
访问 DVWA 登录页面 http://127.0.0.1/dvwa/login.php
输入用户名 admin 和密码 password，点击 Login
Burp 会拦截到请求，在 Proxy → Intercept 中可以看到这个请求
请求的第一行是 POST /dvwa/login.php HTTP/1.1，表示这是一个 POST 请求，登录信息放在请求体中
在 Raw 视图里往下翻，可以看到请求体中的 username=admin&password=password&Login=Login
右键点击这个请求 → 发送到 Repeater，就可以在 Repeater 中反复修改和发送，不用每次都重新登录
2. 改 SQL 注入参数的操作步骤
抓 SQL 注入请求的流程是：
确保 Burp 拦截开关是 Intercept is on
在浏览器中进入 DVWA 的 SQL Injection 页面
在输入框中输入 1，点击 Submit
Burp 会拦截到一个 GET 请求，因为参数是拼在 URL 里面的
在请求的 URL 中找到 id=1 这个参数
把它改成 id=1' or '1'='1' --（注意 -- 后面有一个空格）
点击 Forward 放行
回到浏览器，页面应该返回多个用户的信息（admin、gordonb、1337、pablo、smithy 等）
这个操作的核心是：不管前端输入框怎么限制，Burp 都可以在中间拦截并修改请求，把恶意代码发给服务器。

四、核心收获
1. SQL 注入的本质是改变原 SQL 语句的逻辑
2. `'` 用于闭合字符串，`-- ` 用于注释
3. `or '1'='1'` 构造恒真条件
4. GET 参数在 URL，POST 参数在请求体
5. Burp 可以拦截并修改任何请求
   
五、待解决问题
1.Burp 抓包时总出现系统后台请求怎么办？
问题表现：拦截到的第一个请求经常是 ipv6.msftncsi.com 或类似地址，不是 DVWA 的请求
原因：Windows 系统会在后台自动发送网络连通性测试，这些请求也会经过代理，被 Burp 拦截
解决办法：把这些系统请求点 Forward 放行，直到出现 DVWA 的请求；或者直接关闭拦截（Intercept is off），访问完 DVWA 后在 HTTP 历史记录里找登录请求
2.为什么登录请求是 POST，而 SQL 注入请求是 GET？
POST 用于向服务器提交数据（如用户名密码），参数放在请求体中，浏览器地址栏看不到
GET 用于向服务器请求页面，参数拼在 URL 后面，在地址栏能直接看到
DVWA 的 SQL Injection 模块用的是 GET 方式提交 User ID，所以参数在 URL 里，用 Burp 修改时直接在 URL 上改就行
3.在 Burp 里修改参数时，-- 后面的空格重要吗？
重要。SQL 中 -- 是注释符，但必须后面跟一个空格才能生效。如果没加空格，注释可能不生效，注入语句可能报错。在 Burp 里修改时，记得在 -- 后面敲一个空格。
4.为什么有时候注入成功，有时候没反应？
常见原因：检查闭合是否正确，DVWA 的安全级别是不是设成了 Low，或者语句里有没有语法错误（比如单引号没配对）
5.Burp 的 Intercept 开关和 HTTP 历史记录有什么区别？
Intercept is on：请求会被拦住，你必须手动点 Forward 或 Drop 才能继续，适合精细修改单个请求
HTTP 历史记录：无论拦截开关是开还是关，所有经过代理的请求都会被记录下来，适合事后查看
